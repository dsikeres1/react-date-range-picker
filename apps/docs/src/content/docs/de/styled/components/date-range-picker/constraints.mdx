# Einschränkungen

## Min. & Max. Tage

Beschränken Sie die Länge des Zeitraums mit `minDays` und `maxDays` (inklusiv). Benutzer können keinen Zeitraum bestätigen, der kürzer als `minDays` oder länger als `maxDays` ist.

<CodeDemo source={MinMaxDaysSource}>
  <ExampleWrapper variant="styled" client:load>
    <MinMaxDays client:load />
  </ExampleWrapper>
</CodeDemo>

## Deaktivierte Daten

Verwenden Sie `isDateUnavailable`, um bestimmte Daten dynamisch zu deaktivieren. Deaktivierte Daten können nicht als Anfang oder Ende eines Zeitraums ausgewählt werden.

<CodeDemo source={DisabledDatesSource}>
  <ExampleWrapper variant="styled" client:load>
    <DisabledDates client:load />
  </ExampleWrapper>
</CodeDemo>

## Einzelnes Datum erlauben

Standardmäßig ist ein Zeitraum erlaubt, bei dem Anfang und Ende identisch sind. Setzen Sie `allowSingleDateInRange={false}`, um mindestens zwei unterschiedliche Daten zu erfordern.
