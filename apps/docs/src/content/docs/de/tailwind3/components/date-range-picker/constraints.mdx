# Einschränkungen

## Min & Max Tage

Beschränken Sie die Länge des Bereichs mit `minDays` und `maxDays` (inklusiv). Benutzer können keinen Bereich bestätigen, der kürzer als `minDays` oder länger als `maxDays` ist.

<CodeDemo source={MinMaxDaysSource}>
  <ExampleWrapper variant="tailwind3" client:visible>
    <MinMaxDays client:visible />
  </ExampleWrapper>
</CodeDemo>

## Deaktivierte Daten

Verwenden Sie `isDateUnavailable`, um bestimmte Daten dynamisch zu deaktivieren. Deaktivierte Daten können nicht als Anfang oder Ende eines Bereichs ausgewählt werden.

<CodeDemo source={DisabledDatesSource}>
  <ExampleWrapper variant="tailwind3" client:visible>
    <DisabledDates client:visible />
  </ExampleWrapper>
</CodeDemo>

## Einzelnes Datum erlauben

Standardmäßig ist ein Bereich erlaubt, bei dem der Anfang dem Ende entspricht. Setzen Sie `allowSingleDateInRange={false}`, um mindestens zwei unterschiedliche Daten zu erfordern.

</CodeDemo>
