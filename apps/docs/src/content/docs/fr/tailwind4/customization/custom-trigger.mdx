# Déclencheur Personnalisé

Remplacez le bouton de déclenchement par défaut par votre propre élément en utilisant le render prop `Trigger`. Lorsque vous passez une fonction en tant qu'enfant à `DatePicker.Trigger`, vous recevez l'état et des callbacks pour construire n'importe quelle interface utilisateur de déclenchement que vous souhaitez.

    <CompoundCustomTrigger client:visible />
  </ExampleWrapper>
</CodeDemo>

## Render Props du Déclencheur

| Prop           | Type             | Description                                |
| -------------- | ---------------- | ------------------------------------------ |
| `displayValue` | `string`         | Date sélectionnée formatée                 |
| `placeholder`  | `string`         | Texte du placeholder                       |
| `hasValue`     | `boolean`        | Si une date est sélectionnée               |
| `isOpen`       | `boolean`        | Si la popup est ouverte                    |
| `onToggle`     | `() => void`     | Basculer l'ouverture/fermeture de la popup |
| `onClear`      | `() => void`     | Effacer la sélection                       |
| `locale`       | `Locale`         | Objet de la locale actuelle                |
| `triggerRef`   | `(node) => void` | Callback de ref pour le positionnement     |
