## Remplacement des classes

Chaque partie du composant accepte une prop `className`. Le paquet Tailwind v4 utilise `cn()` (un utilitaire `clsx` + `twMerge`) pour fusionner vos classes avec celles par défaut, afin que les utilitaires en conflit soient résolus correctement :

```tsx
import { DatePicker } from "react-date-range-picker-tailwind4";

<DatePicker.Root value={value} onChange={setValue}>
  {/* Remplacer la largeur et le remplissage du déclencheur */}
  <DatePicker.Trigger className="min-w-[300px] px-4 py-3" />
  <DatePicker.Content className="rounded-2xl shadow-xl">
    <DatePicker.Header>
      <DatePicker.PrevButton />
      <DatePicker.Title className="text-lg" />
      <DatePicker.NextButton />
    </DatePicker.Header>
    <DatePicker.Grid />
  </DatePicker.Content>
</DatePicker.Root>;
```
