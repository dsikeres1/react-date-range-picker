## Sobrescrevendo Classes

Cada parte composta aceita uma prop `className`. O pacote Tailwind v4 usa `cn()` (um utilitário `clsx` + `twMerge`) para mesclar suas classes com as padrões, para que os utilitários conflitantes sejam resolvidos corretamente:

```tsx
import { DatePicker } from "react-date-range-picker-tailwind4";

<DatePicker.Root value={value} onChange={setValue}>
  {/* Override trigger width and padding */}
  <DatePicker.Trigger className="min-w-[300px] px-4 py-3" />
  <DatePicker.Content className="rounded-2xl shadow-xl">
    <DatePicker.Header>
      <DatePicker.PrevButton />
      <DatePicker.Title className="text-lg" />
      <DatePicker.NextButton />
    </DatePicker.Header>
    <DatePicker.Grid />
  </DatePicker.Content>
</DatePicker.Root>;
```
