# Restrições

## Dias Mínimos e Máximos

Restrinja a duração do intervalo com `minDays` e `maxDays` (inclusivo). Os usuários não podem confirmar um intervalo menor que `minDays` ou maior que `maxDays`.

<CodeDemo source={MinMaxDaysSource}>
  <ExampleWrapper variant="tailwind4" client:load>
    <MinMaxDays client:load />
  </ExampleWrapper>
</CodeDemo>

## Datas Desabilitadas

Use `isDateUnavailable` para desabilitar dinamicamente datas específicas. Datas desabilitadas não podem ser selecionadas como início ou fim de um intervalo.

<CodeDemo source={DisabledDatesSource}>
  <ExampleWrapper variant="tailwind4" client:load>
    <DisabledDates client:load />
  </ExampleWrapper>
</CodeDemo>

## Permitir Data Única

Por padrão, um intervalo onde o início é igual ao fim é permitido. Defina `allowSingleDateInRange={false}` para exigir pelo menos duas datas distintas.

</CodeDemo>
