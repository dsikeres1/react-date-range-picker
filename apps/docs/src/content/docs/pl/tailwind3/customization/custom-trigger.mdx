# Własny Wyzwalacz

Zastąp domyślny wyzwalacz własnym elementem, używając render prop. Komponent `Trigger` akceptuje jako dziecko funkcję, która otrzymuje stan i callbacki, pozwalając na renderowanie dowolnego niestandardowego przycisku lub pola wejściowego.

    <CompoundCustomTrigger client:visible />
  </ExampleWrapper>
</CodeDemo>

## Render Props Wyzwalacza

| Prop           | Typ              | Opis                                 |
| -------------- | ---------------- | ------------------------------------ |
| `displayValue` | `string`         | Sformatowana wybrana data            |
| `placeholder`  | `string`         | Tekst zastępczy                      |
| `hasValue`     | `boolean`        | Czy data jest wybrana                |
| `isOpen`       | `boolean`        | Czy okienko jest otwarte             |
| `onToggle`     | `() => void`     | Przełącz otwarcie/zamknięcie okienka |
| `onClear`      | `() => void`     | Wyczyść zaznaczenie                  |
| `locale`       | `Locale`         | Bieżący obiekt locale                |
| `triggerRef`   | `(node) => void` | Callback ref do pozycjonowania       |
