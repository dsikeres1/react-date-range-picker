# Rozmiary

Pakiet `styled` zawiera 4 wbudowane warianty rozmiarów, które proporcjonalnie skalują typografię, odstępy, wymiary komórek i wymiary wyzwalacza.

## Dostępne rozmiary

| Rozmiar   | Rozmiar komórki | Podstawa czcionki | Min. szerokość wyzwalacza |
| --------- | --------------- | ----------------- | ------------------------- |
| `small`   | 28px            | 11px              | 160px                     |
| `medium`  | 36px            | 13px              | 200px                     |
| `large`   | 44px            | 14px              | 240px                     |
| `x-large` | 52px            | 15px              | 280px                     |

`medium` jest domyślny.

## Użycie

Przekaż prop `size` do dowolnego komponentu wyboru:

<CodeDemo source={SizesSource}>
  <ExampleWrapper variant="styled" client:load>
    <Sizes client:load />
  </ExampleWrapper>
</CodeDemo>

## Niestandardowe rozmiary

Stwórz własny rozmiar, definiując klasę CSS, która nadpisuje zmienne rozmiaru:

```css
.rdrp-size-custom {
  --rdrp-cell-size: 40px;
  --rdrp-nav-btn-size: 30px;
  --rdrp-font-size-base: 14px;
  --rdrp-font-size-md: 15px;
  --rdrp-trigger-min-width: 220px;
  --rdrp-popup-padding: 18px;
}
```

Następnie zastosuj go za pomocą wrappera:

```tsx
<div className="rdrp-size-custom">
  <DatePicker value={value} onChange={setValue} />
</div>
```

Lub za pomocą API komponentów złożonych (Compound Component API), dodaj klasę do `Root`:

```tsx
<DatePicker.Root value={value} onChange={setValue} className="rdrp-size-custom">
  {/* ... */}
</DatePicker.Root>
```

Możesz również nadpisać zmienne rozmiaru za pomocą stylów inline:

<Admonition type="tip">
  Rozmiar `medium` jest domyślny i nie stosuje żadnych nadpisań. Podstawowe zmienne w
  `variables.css` to wartości dla rozmiaru `medium`.
</Admonition>
