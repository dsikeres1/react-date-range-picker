# Пользовательский футер

## Изменение порядка в футере

Меняйте порядок кнопок в футере или добавляйте новые, например `TodayButton`. Поскольку каждая кнопка является независимой составной частью, вы контролируете порядок и компоновку.

## Переопределение классов

Каждая составная часть принимает свойство `className`. Пакет для Tailwind v4 использует `cn()` (утилиту, объединяющую `clsx` и `twMerge`) для слияния ваших классов со стандартными, поэтому конфликтующие утилиты разрешаются корректно:

```tsx
import { DatePicker } from "react-date-range-picker-tailwind4";

<DatePicker.Root value={value} onChange={setValue}>
  {/* Override trigger width and padding */}
  <DatePicker.Trigger className="min-w-[300px] px-4 py-3" />
  <DatePicker.Content className="rounded-2xl shadow-xl">
    <DatePicker.Header>
      <DatePicker.PrevButton />
      <DatePicker.Title className="text-lg" />
      <DatePicker.NextButton />
    </DatePicker.Header>
    <DatePicker.Grid />
  </DatePicker.Content>
</DatePicker.Root>;
```
