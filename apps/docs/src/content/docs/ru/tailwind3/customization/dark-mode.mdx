# Темный режим

Пакет Tailwind v3 включает полную поддержку темного режима с использованием варианта `dark:` от Tailwind. Каждый элемент компонента имеет явные классы для темного режима.

## Как это работает

Тема Tailwind v3 использует конкретные классы цветов с вариантами `dark:` повсеместно. Например:

```
bg-white dark:bg-slate-950
text-slate-900 dark:text-slate-50
border-slate-200 dark:border-slate-800
```

Темный режим активируется, когда срабатывает вариант `dark:` от Tailwind, что зависит от стратегии `darkMode` в вашей конфигурации Tailwind.

## Конфигурация Tailwind

### Стратегия с классом (рекомендуется)

Установите `darkMode: "class"` в вашей конфигурации Tailwind. Темный режим активируется, когда класс `dark` присутствует на `<html>` или родительском элементе:

```js
// tailwind.config.js
module.exports = {
  darkMode: "class",
  content: [
    "./src/**/*.{js,ts,jsx,tsx}",
    "./node_modules/react-date-range-picker-tailwind3/dist/**/*.{js,mjs}",
  ],
};
```

Переключение темного режима:

```tsx
function App() {
  const [dark, setDark] = useState(false);

  return (
    <div className={dark ? "dark" : ""}>
      <button onClick={() => setDark(!dark)}>Переключить тему</button>
      <DatePicker value={value} onChange={setValue} />
    </div>
  );
}
```

Или примените класс к `<html>`:

```tsx
document.documentElement.classList.toggle("dark", isDark);
```

### Стратегия с селектором

Tailwind v3.4+ поддерживает `darkMode: "selector"`, что позволяет использовать `[data-theme="dark"]` или любой другой пользовательский селектор:

```js
// tailwind.config.js
module.exports = {
  darkMode: ["selector", '[data-theme="dark"]'],
  content: [
    "./src/**/*.{js,ts,jsx,tsx}",
    "./node_modules/react-date-range-picker-tailwind3/dist/**/*.{js,mjs}",
  ],
};
```

```tsx
<div data-theme={dark ? "dark" : "light"}>
  <DatePicker value={value} onChange={setValue} />
</div>
```

### Медиа-стратегия

Установите `darkMode: "media"` (или опустите — это значение по умолчанию), чтобы следовать настройкам ОС:

```js
// tailwind.config.js
module.exports = {
  darkMode: "media",
  // ...
};
```

Дополнительная настройка не требуется. Компонент будет следовать `prefers-color-scheme`.

## Пары цветов

Вот пары цветов для светлого/темного режимов, используемые в компоненте:

| Элемент                | Светлый             | Темный                       |
| ---------------------- | ------------------- | ---------------------------- |
| Фон                    | `bg-white`          | `dark:bg-slate-950`          |
| Текст                  | `text-slate-900`    | `dark:text-slate-50`         |
| Приглушенный текст     | `text-slate-500`    | `dark:text-slate-400`        |
| Граница                | `border-slate-200`  | `dark:border-slate-800`      |
| Фон при наведении      | `bg-slate-100`      | `dark:bg-slate-800`          |
| Основной цвет          | `bg-sky-500`        | `dark:bg-sky-500`            |
| Основной текст         | `text-sky-500`      | `dark:text-sky-400`          |
| Фон диапазона          | `bg-sky-50`         | `dark:bg-sky-950/50`         |
| Диапазон при наведении | `bg-sky-50/50`      | `dark:bg-sky-950/30`         |
| Кольцо фокуса          | `ring-sky-500`      | `ring-sky-500`               |
| Смещение кольца        | `ring-offset-white` | `dark:ring-offset-slate-950` |

## Интеграция с Docusaurus

Если вы используете Docusaurus (который использует `data-theme` на `<html>`), настройте стратегию селектора в Tailwind:

```js
// tailwind.config.js
module.exports = {
  darkMode: ["selector", '[data-theme="dark"]'],
  // ...
};
```

Docusaurus устанавливает `<html data-theme="dark">`, когда активен темный режим, и вариант `dark:` от Tailwind будет срабатывать по этому селектору.

## Интеграция с next-themes

```tsx
// app/layout.tsx
import { ThemeProvider } from "next-themes";

export default function RootLayout({ children }) {
  return (
    <html suppressHydrationWarning>
      <body>
        <ThemeProvider attribute="class" defaultTheme="system">
          {children}
        </ThemeProvider>
      </body>
    </html>
  );
}
```

`next-themes` добавляет/удаляет класс `dark` на `<html>`. С `darkMode: "class"` в вашей конфигурации Tailwind, компонент переключается автоматически.

## Принудительное включение режима

Чтобы принудительно заставить компонент всегда использовать светлый или темный режим, оберните его в контейнер с соответствующим классом:

```tsx
{
  /* Всегда светлый — нет родительского элемента с классом "dark" */
}
<div className="">
  <DatePicker value={value} onChange={setValue} />
</div>;

{
  /* Всегда темный */
}
<div className="dark">
  <DatePicker value={value} onChange={setValue} />
</div>;
```

<Admonition type="note">
При использовании `darkMode: "class"`, класс `dark` должен быть на родительском элементе или `<html>`. Сам компонент выбора даты не управляет переключением темного режима.
</Admonition>
