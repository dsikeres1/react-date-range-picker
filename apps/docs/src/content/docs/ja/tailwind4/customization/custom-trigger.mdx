# カスタムトリガー

`Trigger`レンダープロップを使用して、デフォルトのトリガーボタンを独自の要素に置き換えます。`DatePicker.Trigger`の子として関数を渡すと、状態とコールバックを受け取り、自由なトリガーUIを構築できます。

    <CompoundCustomTrigger client:visible />
  </ExampleWrapper>
</CodeDemo>

## トリガーレンダープロップ

| プロップ       | 型               | 説明                             |
| -------------- | ---------------- | -------------------------------- |
| `displayValue` | `string`         | フォーマットされた選択日         |
| `placeholder`  | `string`         | プレースホルダーテキスト         |
| `hasValue`     | `boolean`        | 日付が選択されているかどうか     |
| `isOpen`       | `boolean`        | ポップアップが開いているかどうか |
| `onToggle`     | `() => void`     | ポップアップの開閉をトグルします |
| `onClear`      | `() => void`     | 選択をクリアします               |
| `locale`       | `Locale`         | 現在のロケールオブジェクト       |
| `triggerRef`   | `(node) => void` | 配置用のrefコールバック          |
