# カスタムトリガー

レンダープロップを使用して、デフォルトのトリガーを独自の要素に置き換えます。`Trigger`コンポーネントは、状態とコールバックを受け取る関数を子として受け入れ、任意のカスタムボタンや入力をレンダリングできます。

  <ExampleWrapper variant="tailwind3" client:load>
    <CompoundCustomTrigger client:load />
  </ExampleWrapper>
</CodeDemo>

## トリガーのレンダープロップ

| プロップ       | 型               | 説明                             |
| -------------- | ---------------- | -------------------------------- |
| `displayValue` | `string`         | フォーマットされた選択日         |
| `placeholder`  | `string`         | プレースホルダーテキスト         |
| `hasValue`     | `boolean`        | 日付が選択されているかどうか     |
| `isOpen`       | `boolean`        | ポップアップが開いているかどうか |
| `onToggle`     | `() => void`     | ポップアップの開閉をトグルする   |
| `onClear`      | `() => void`     | 選択をクリアする                 |
| `locale`       | `Locale`         | 現在のロケールオブジェクト       |
| `triggerRef`   | `(node) => void` | 配置のためのrefコールバック      |
