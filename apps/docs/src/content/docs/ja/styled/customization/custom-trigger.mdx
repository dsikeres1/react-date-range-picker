# カスタムトリガー

デフォルトのトリガーを独自のボタンや入力に置き換えます。`Trigger`コンポーネントは、トリガーの外観と動作を完全に制御できるレンダープロップを受け入れます。

  <ExampleWrapper variant="styled" client:visible>
    <CompoundCustomTrigger client:visible />
  </ExampleWrapper>
</CodeDemo>

## トリガーのレンダープロップ

| プロップ       | 型               | 説明                             |
| -------------- | ---------------- | -------------------------------- |
| `displayValue` | `string`         | フォーマットされた選択日         |
| `placeholder`  | `string`         | プレースホルダーテキスト         |
| `hasValue`     | `boolean`        | 日付が選択されているかどうか     |
| `isOpen`       | `boolean`        | ポップアップが開いているかどうか |
| `onToggle`     | `() => void`     | ポップアップの開閉を切り替える   |
| `onClear`      | `() => void`     | 選択をクリアする                 |
| `locale`       | `Locale`         | 現在のロケールオブジェクト       |
| `triggerRef`   | `(node) => void` | 位置決めのためのrefコールバック  |
